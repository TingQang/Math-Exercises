# 解决行内公式极限符号问题与测试章节重组计划

## 一、问题分析

### 1. 行内公式极限符号问题
- 行内公式中，极限符号的下标默认在右侧，不是正下方，这是LaTeX默认行为
- 要让极限符号下标在正下方，需要使用`\limits`命令

### 2. 测试章节结构问题
- 当前测试章节按照具体知识点组织，不符合设计文档要求
- 需要按照"高等数学"、"线性代数"和"概率论与数理统计"进行分类

### 3. make工具配置问题
- 需要安装GNU Make工具
- 配置Makefile，使其与latexmk具有相同的编译选项

### 4. 编译输出目录问题
- 需要确保所有编译命令将输出目录设置为./build/

## 二、解决方案

### 1. 解决行内公式极限符号问题
- 修改`config/commands.tex`文件，将极限快捷命令中的`\limits`保留或添加
- 确保行内公式中使用`\lim\limits_{x \to a}`格式显示

### 2. 重新组织测试章节
- 将当前测试章节内容按照学科分类重组
- 高等数学：极限与连续、导数与微分、积分、微分方程
- 线性代数：行列式、矩阵、向量与线性空间、线性方程组、特征值与特征向量
- 概率论与数理统计：随机事件与概率、随机变量及其分布、多维随机变量及其分布、大数定律与中心极限定理、统计量及其分布

### 3. 配置make工具
- 安装GNU Make工具
- 创建或更新Makefile，配置与latexmk相同的编译选项
- 确保Makefile支持清理编译产物、自动编译等功能

### 4. 设置编译输出目录
- 确保所有编译命令将输出目录设置为./build/
- 更新.vscode/settings.json中的输出目录配置
- 更新Makefile中的输出目录配置

## 三、实施步骤

1. **修改极限命令定义**
   - 文件：`config/commands.tex`
   - 将所有极限快捷命令修改为使用`\limits`，如`\newcommand{\limx0}{\lim\limits_{x \to 0}}`

2. **重新组织测试章节**
   - 文件：`content/ch02.tex`
   - 按照学科分类重组内容
   - 确保每个学科下的内容结构符合设计文档要求

3. **安装并配置make工具**
   - 安装GNU Make工具
   - 更新Makefile，添加与latexmk相同的编译选项
   - 确保Makefile支持清理编译产物

4. **设置编译输出目录**
   - 更新.vscode/settings.json中的输出目录配置
   - 更新Makefile中的输出目录配置
   - 确保所有编译命令使用相同的输出目录

5. **验证编译结果**
   - 使用make命令编译文档
   - 检查输出目录是否正确
   - 检查行内公式极限符号是否显示正确
   - 检查测试章节结构是否符合要求

## 四、预期效果

- 行内公式中的极限符号下标显示在正下方
- 测试章节按照学科分类重新组织，符合设计文档要求
- make工具安装并配置完成，与latexmk具有相同的编译选项
- 所有编译产物输出到./build/目录
- 编译过程顺利，没有错误或警告

## 五、注意事项

- 修改极限命令定义时，确保所有相关文件都更新
- 重新组织测试章节时，确保所有链接和引用都正确
- 安装make工具时，选择合适的版本和安装方式
- 配置编译输出目录时，确保所有相关配置都一致
